Prodrive Technologies Dev Challenge 2019

Rules

  • 2 players
  • play board (11 by 11 grid)
  • turn based (3 seconds per turn)

During its turn the player places one piece on an empty space in the grid. A player has 3 seconds to perform its turn. If the time limit is not met, the player automatically loses the game. Player 1 begins and the game ends when the grid is completely filled.

The goal is to create the longest group of pieces that are horizontally or vertically connected. The length of a group is determined by the shortest route from the two points that are the furthest apart in the group, also called the graph diameter (http://mathworld.wolfram.com/GraphDiameter.html). For example in the grid below the longest group for x has a length of 6 and for o it has a length of 4. The results for o will be displayed like: 1-1-0-1 which means the player has 1 group of length one, 1 group of two, 0 of three and 1 of four.

x|o|o|x|o|o|
x|x|x|x|o|o|
x|x|x|o|x|o|

The player with the longest group wins the game. If the longest group of both players has the same length then the amount of these groups determines the winner, if this amount is also the same then the amount of groups with 1 shorter length etc. If all of these are the same then it is a draw. Since the starting player has the advantage of the first turn, player 2 receives a bonus group with the length of 1 shorter than its longest group.

In the following example Player 1 places + signs and player 2 places o signs. The result of the game looks like this:

  • Blue: 1 group of 17 long, 6 groups of 4 long, etc.
  • Red: 1 group of 7 long, 2 groups 6 long (BONUS), 2 groups of 5 long, etc.

Scoring

The scores for each game are awarded as follows:

  • Win 2pt
  • Draw 1pt
  • Lose 0pt

The winner of the tournament is the player with the highest score.

Game host

The game is hosted by an executable, the executable enforces the rules and ensures both player programs have the same amount of computer resources available. The following restrictions exist:

  • The automated program is not allowed to start child processes.
  • The automated program has a memory limit of 1GB.
  • The automated program receives a CPU affinity such that it has access to 1 CPU core.
  • Only the automated program whose turn it is can use the CPU. The other process is suspended.
  • The automated program is only allowed to read files from its startup directory.

A program can communicate with the game host by writing to STDOUT and reading from STDIN.

  • At startup the player id (0 or 1) needs to be read from STDIN. (0 stands for player 1 and 1 for player 2)
  • On even turns player 1 can read from STDIN and player 2 on uneven turns. STDIN will contain the x and y coordinate of the move the opposite player made, separated by a space.
  • On uneven turns player 1 can write to STDOUT and player 2 on even turns. The format is x and y coordinate separated by a space.

For example the fist two moves of a game can look like this:

  • Player 1:
    • STDIN: 1
    • STDOUT: 0 0
    • STDIN: 0 1
  • Player 2:
    • STDIN: 2
    • STDIN: 0 0
    • STDOUT: 0 1

This results in a grid that looks like this:

1|2

The game host can be started with the following command: gamehost.exe C:\path\to\payer1AI.exe "" C:\path\to\payer2AI.exe "". An optional -s <number> flag can be given to enable the sandbox, the number indicates the CPU affinity. Please ensure your AI works with the sandbox flag! Optional command line parameters can be given to each executable, or just empty quotes if you don't want to provide any. For example, if you want to execute a .jar file the command can look like: gamehost.exe payer1AI.exe "" java.exe "-jar player2AI.jar".

Hand-outs

Each team will receive a USB drive containing the following:

  • The source code and executable files of three example automated programs. ExampleAI is an C++ program, ExampleAI-CS is a C# program and JavaExampleAI is a Java program. These examples produce random moves and demonstrate communication with the game host.
  • The game host executable which can be used for testing.
  • A Java executable which is needed to run the Java example. java.exe should either be in the same folder as the game host or the exact path should be specified. This rule holds for any executable which is passed to the game host.

Submission

All solutions must be submitted before the deadline. Any submissions after that, will not be considered.

The source code of all solutions must be submitted, together with instructions on how to compile it. If you already have a compiled executable file, please include that as well. The supported compilers are: C++, C#, Python, Java. If you'd like to use a different language, please submit a compiled version of your program.
